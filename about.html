The Drag Package Manager

Whispix is a minimal, independent, source-based Linux distribution that uses drag to manage it's packages. Drag is written in Bash with the purpose of delivering a fast and customizable set of tools. It fetches PKGBUILDs from Arch Linux's repositories and the AUR to maximize package support. Drag makes editing PKGBUILDs easy in order to put the user in control of their system. Drag features file tracking for installed packages, allowing for a clean removal process. Unlike most package managers, drag does not support automatic dependency resolution, though it does possess the ability list the recommended dependencies found in the PKGBUILDs.

System Requirements

Because Whispix is a source distribution that is built from scratch, you must be comfortable with your computer specifications. Building Whispix on old hardware could take quite some time due to large packages like glibc and gcc. Whispix does not rely on a singular processor type, making Whispix a universal distribution. The bootstrap follows a similar path to LFS. This also means that by default it compiles glibc and other gnu tools. If you are opposed to this or any other aspect of the bootstrap, feel free to edit it and make it your own. Any help and suggestions for Whispix are greatly appreciated.

Initial Install

Whispix compiles every package from scratch very similarly to LFS. Though thankfully, unlike LFS, much of it is automated by the bootstrap. The expected product of an initial install, unless customized further, would be a very basic Linux system that doesn't even include a traditional init system. The standard Whispix installation provides a minimal shell script at /sbin/init to start necessary services. While surprising to some, this shell script is still very capable of running a real desktop environment. Currently, I am writing this using my daily system (running Whispix) configured to run modern applications such as Firefox, OBS, GIMP, Blender and others.

Comparisons

Though I haven't given my package manager a true benchmark to compare to other distributions, from personal experience I can say as a former Gentoo user that my drag package manager does feel faster than Gentoo's portage when compiling, though keep in mind that this does not include dependency resolution. This means that the time it takes to install large sets of packages may very well take longer than you would expect on Gentoo since you must decide yourself which dependencies you wish to install. I personally prefer this, since I feel that I have more power and fine-tuned control over my system now. Gentoo's USE flags are genius and extremely efficient, but the fundamental concept begs you to consider that the developers must create these flags themselves and that not every possible USE flag for every possible package is going to be considered, therefore proving drag a more customizable package manager. Gentoo is nevertheless an extremely powerful and refined distribution, but Whispix does offer more fine-tuned control over your system if that is what you are looking for.

Is Whispix Right For You?

There is no "best" distribution. Each distribution has its strengths and picking the right one for you should be solely because of your personal preferences and needs in a system. Like many other advanced distributions, Whispix is definitely not for the average Linux user. Whispix is meant for people who have a deep understanding of Linux systems and manually compiling packages to suit their needs. Compiling packages, even with the default PKGBUILDs downloaded from Arch, will challenge you. You will run into all sorts of compilation, configuration, and run-time errors. If you struggle to compile or get packages to work on Gentoo or other source-based distributions, you will almost definitely struggle harder on Whispix. Now for those who understand how to compile and configure packages manually for your system, you may very well find Whispix to be just what you've been needing.
